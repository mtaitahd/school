# Child Registration & Parent Claiming Flow

## 1. Child Registration (by Teacher)

Only **teachers** can create learner accounts.

### Step-by-Step

1. Teacher logs in в†’ session with `role='teacher'`
2. Goes to `teacher/learners.php?add=1`
3. Fills form: **username**, **first_name**, **last_name**, **password**, **phone**, **parent_phone**
4. POSTs to `teacher/student-actions.php?action=add_student`

### What the System Does

| Step | Action |
|------|--------|
| Validate | Checks required fields, ensures username unique in `users` table |
| Hash password | `password_hash($password, PASSWORD_DEFAULT)` (bcrypt) |
| Generate claim code | `ClaimCodeGenerator::generateCode()` в†’ format `KH-XXXXXX` (e.g., `KH-7F92K1`) |
| Generate access code | `strtoupper(substr(md5(uniqid(...)), 0, 8))` в†’ 8-char uppercase alphanumeric |
| Insert user | `INSERT INTO users` with `role='learner'`, claim_code, parent_phone, etc. |
| Insert access code | `INSERT INTO student_access_codes` (student_id, teacher_id, access_code) |
| Enroll in class | If `class_id` provided в†’ `INSERT INTO class_enrollments` |
| Send SMS | If `parent_phone` provided в†’ SMS with claim code to parent |
| Redirect | Back to `learners.php?success=1&code=KH-XXXXXX` |

### Two Codes Generated Per Learner

| Code | Format | Purpose | Table |
|------|--------|---------|-------|
| **Claim Code** | `KH-XXXXXX` | Parent links child to dashboard | `users.claim_code` |
| **Access Code** | 8 uppercase chars | Legacy/secondary linking | `student_access_codes.access_code` |

## 2. SMS Service (`php/sms_service.php`)

### Provider
- **Webline Africa** (`https://sms.webline.africa/api/v3/sms/send`)
- Auth: Bearer token from `.env` (`SMS_API_TOKEN`)
- Sender ID: `TAARIFA`

### Message Flow
1. Validate phone (Tanzanian format `+255[6-7]XXXXXXXX`)
2. Rate limit: 1s delay between sends
3. Log to `sms_logs` with status `pending`
4. POST JSON to API with retry (3 attempts, exponential backoff)
5. Update `sms_logs` status в†’ `sent` or `failed`

### SMS Types & Triggers

| Type | Trigger | Message Content |
|------|---------|-----------------|
| **parent_link** | Child created (if parent_phone provided) | "Your child account has been created. Use claim code: KH-XXXXXX to connect to your parent dashboard" |
| **parent_link** | Teacher resends SMS | Same as above |
| **parent_link** | Parent successfully claims child | "Umefanikiwa kuunganisha akaunti yako na mtoto wako [name]" (Swahili confirmation) |
| **performance** | Student result recorded | "Mtoto wako [name] amepata [result] kwenye mtihani" |
| **fee_payment** | Fee payment confirmation | "Malipo ya shule kwa [name] yamepokelewa" |
| **assignment** | Assignment created | "Mtoto wako [name] ana kazi [title] inayohitaji kukamilishwa kabla ya [dueDate]" |

## 3. Parent Claiming Flow

### Prerequisite
Parent must have an account (register via `register.php` with `role='parent'`) and be logged in.

### Method A: Claim Code (Primary)

1. Parent logs in в†’ `parent/dashboard.php`
2. If no children linked, sees "Claim Child" button
3. Opens modal, enters `KH-XXXXXX` code
4. POSTs to `parent/claim-child.php`

**System processing:**
```
1. Validate format: regex /^KH-[A-Z0-9]{6}$/
2. Lookup student: getCodeInfo() queries users table
3. Check not already claimed: isClaimed() or parent_student_links
4. UPDATE users SET parent_claimed=1, parent_id=? WHERE claim_code=?
5. INSERT INTO parent_student_links (parent_id, student_id)
6. Send confirmation SMS to parent phone
7. Redirect to dashboard.php
```

### Method B: Access Code (Legacy)

1. Parent goes to `parent/claim-student.php`
2. Enters 8-character access code
3. System looks up in `student_access_codes` (checks `is_active=1` and `expires_at`)
4. If valid в†’ `INSERT INTO parent_student_links`
5. Send confirmation SMS

### Comparison

| Feature | Claim Code | Access Code |
|---------|-----------|-------------|
| Format | `KH-XXXXXX` | 8 uppercase chars |
| Generated by | `random_int()` (crypto-safe) | `md5(uniqid())` |
| Stored in | `users.claim_code` | `student_access_codes.access_code` |
| Has expiry | No | Yes (30 days) |
| Updates `users.parent_claimed` | Yes | No |

---

## 4. Learner Login

Unique: learners do **not** need a password.

1. Goes to `learner/login.php`
2. Enters **only username**
3. System: `SELECT * FROM users WHERE username=? AND role='learner' AND is_active=1`
4. If found в†’ creates session (`user_id`, `username`, `role`, `first_name`, `last_name`)
5. Redirects to `learner/dashboard.php`

---

## 5. Key Database Tables

| Table | Role |
|-------|------|
| `users` | All users (admins, teachers, parents, learners). Contains `claim_code`, `parent_claimed`, `parent_id`, `parent_phone` for linking |
| `student_access_codes` | 8-char access codes per student, with expiry |
| `parent_student_links` | Links parent в†’ student (supports multiple parents per child) |
| `class_enrollments` | Which students belong to which classes |
| `sms_logs` | All SMS attempts, statuses, and API responses |
| `notifications` | In-app notifications for parents/teachers |
